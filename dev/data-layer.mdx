title: 'Data layer'

---

`data/site-content.ts` is the single source of truth for every string, label, and structured text object rendered on the site. Components import directly from this file — no API calls, no CMS, no environment variables. To change any visible text, edit the relevant export here.

<Note>
  You do not need to restart the Next.js dev server after editing `site-content.ts`. Hot module replacement picks up the change automatically.
</Note>

## Exported constants

## Updating content

All visible text on the site is controlled by this file. To make a change:

<Steps>
  <Step title="Open the file">
    Open `data/site-content.ts` in your editor.
  </Step>
  <Step title="Find the relevant export">
    Locate the exported constant that corresponds to the section you want to change. Each export maps directly to a named page section.
  </Step>
  <Step title="Edit the value">
    Update the string or object field. Do not change the property names — components depend on the exact shape.
  </Step>
  <Step title="Save and verify">
    Save the file. The Next.js dev server reloads automatically. Confirm the change in the browser.
  </Step>
</Steps>

<Warning>
  Do not rename exported constants or remove object properties. Components reference these by name and will break silently if a field is missing.
</Warning>
